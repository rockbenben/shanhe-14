# 浮生长卷 · floating-life

> 365 开源计划 #021 · 编排式叙事框架 — 作者编排故事，引擎负责推进与记忆

千世书（#015）的镜像姐妹作：千世书用数值涌现故事，浮生长卷用编排承载故事。
故事是数据：线性章节串（珠）、章内分支章末汇聚（DAG）、选择只落 flag、flag 驱动回响与结局。

首发故事 **《山河十四年》**：1931 九一八 → 1945 胜利 · 据史实与亲历记载改编的互动叙事。
十二章，零配置、无需 API Key，打开即读。

## 《山河十四年》的做法

- **史料为底**：场景尽量取自真实记载（回忆录、当年报刊、口述史），58 处出处注随文可点，尽量附验证过的来源链接
- **档案照片为幕**：40+ 张公有领域档案照片（Wikimedia Commons、美国国会图书馆、美国国家档案馆、法国国家图书馆等），逐张核验许可；事件照对应事件，氛围照如实标注「非该场景实拍」
- **版画插画区隔**：虚构场景用 AI 生成的木刻版画风插画——刻意非写实，与真实照片一望即知地分开
- **环境声底**：每章一段极低音量的公版环境声（暮蝉、火车、江水、风雪……），可一键静音
- **诚实三原则**：查证不到的宁缺；署名必须如实；影像缺席也是记录（第十章章封的取舍见 `public/covers/CREDITS.md`）
- **AI 共创**：框架与叙事编排由 AI 起草、人工定夺；史料检索与微调由 AI 逐拍执行、出处人工抽验；照片与声音由 AI 搜证、人工核验许可

## 体验

简繁一键切换（OpenCC）· 空格/方向键/点击推进 · ←/↑ 回退可重选 · 看全图纯赏画面 · 卷末「回顾全程」与「历史影像」影像志 · 四种结局按选择分化

## 快速开始

```bash
npm install
npm run dev   # http://localhost:5173
npm test
npm run build # 纯静态，dist/ 可部署到任意静态托管（base './'，任意子路径可用）
```

部署提示：上线后把 `index.html` 中 `og:image` / `twitter:image` 换成绝对 URL（爬虫不解析相对路径）。

## 故事格式

见 `src/stories/schema.ts`（zod 定义即文档）：Story → chapters[]（章：题引/史实注/章封照片/环境声）→ beats[]（节点 DAG：叙事/选择/回响/史料注/档案照片）→ endings（按 flag 分化，末项兜底）。校验含引用解析、环检测、可达性、兜底结局唯一性——写错在构建期就会被拦下。

`scripts/` 下的 `_photos_*.json`、`_sound.json` 等是全部影像与声音的搜证清单（含来源页与许可原文），`audit-*.mts` 是内容审计脚本。
