# 浮生长卷：编排式叙事框架 + 首发双故事

日期：2026-07-02
项目：365 开源计划 #021（暂名 `021-floating-life` ／「浮生长卷」）
目标：为知乎人文季打造一个**纯故事性（非数值）**的叙事游戏框架，作者编排故事、引擎负责推进与记忆，首发《山河十四年》（历史单元）。

## 定位

千世书（015）的镜像姐妹作：千世书是**数值涌现故事**（模拟器），021 是**作者编排故事**（叙事卷）。一个框架＋多个故事，故事是数据文件。零 Key 完整可玩——无 Key 玩家看到的是完整、精确编排的故事，不是残缺版；填 Key 后 AI 仅作润色皮层。

参赛叙事：与千世书构成"同一方法论的两极"（数值模拟 × 编排叙事），可分报不同单元（报名前确认活动规则是否允许一人多项目）。

## 故事格式（串珠数据模型）

```
Story
├─ meta：id、题名、题引、封面 art
├─ chapters[]（章＝珠，线性推进，情感曲线锚在章序上）
│   ├─ title / 章引 / 章封 art / 史实注（历史题材专用，可选）
│   └─ beats[]（节点小图：章内分支，章末必汇聚）
│       ├─ narrative（预制文本，无 Key 时的完整体验）
│       ├─ choices[]（text、跳转 beat、flagsSet——无数值）
│       └─ art（稳定 id + gen，沿用千世书配图体系）
├─ flags（选择的记忆，贯穿全书）
└─ finale：尾章按 flag 组合分化的多结局（编排式）
```

**铁律：章内分支、章末汇聚。** 写作量随章数线性增长，这是框架养多个故事的前提。选择的分量靠 flag 在后续章节的回响（某章一句话、尾章走向）体现。schema 用 zod 定义（沿用千世书作法）。

## 引擎与 AI 润色层

- **引擎**（纯本地）：章节推进、beat 图遍历、flag 存取、结局分化、存档续读（localStorage）、全程回顾长卷。
- **AI 皮层**（填 Key 才激活）：
  1. 过场扩写——以预制文本为底稿，按 flag 与人物状态扩写氛围细节；底稿保下限，AI 抬上限；
  2. 自由输入——玩家可在选项外写行动/对白，AI 按「编剧守则」回应（可丰富场面，不得改变本章走向），温柔送回主脊。

## UI 呈现

视觉小说式，继承千世书成熟范式：整屏插画为幕、卷文浮层、章节封面页的翻章仪式感、结局卡＋分享入口页（`/s/<story>/`）。配图走 thousand-lives-art-pipeline。

## 首发故事：《山河十四年》（历史单元）

普通人视角穿过 1931 九一八到 1945 胜利的完整抗战。**主角设定：1931 年沈阳的中学生，流亡入关，辗转成为报馆记者**——记者身份让他有理由抵达各个历史现场而不必成为士兵，天然承载"群像见证者"视角，也呼应"记录"这一人文主题。

章节骨架（12～14 章，每章锚一个真实历史节点，动笔前逐章检索复核史实）：

| 章 | 年份·锚点 | 主角处境（草案） |
|---|---|---|
| 1 | 1931 九一八 | 沈阳一夜，家国骤碎 |
| 2 | 1932 一二八淞沪 | 流亡入关，初抵上海 |
| 3 | 1935 华北危局·一二九 | 北平学潮，笔的启蒙 |
| 4 | 1937 七七卢沟桥 | 全面抗战爆发 |
| 5 | 1937 淞沪会战 | 战地初执记者笔 |
| 6 | 1937.12 南京 | **侧写处理**：随难民西撤，江边回望；暴行经幸存者证言呈现，不作场面直写 |
| 7 | 1938 台儿庄·武汉 | 战时武汉的众生与撤退 |
| 8 | 1939–40 重庆大轰炸 | 雾都防空洞里的日常 |
| 9 | 1941 珍珠港·孤岛终结 | 上海旧识的消息（孤岛调研素材在此复用） |
| 10 | 1942 河南饥荒 | 记者的良心之战（发或不发） |
| 11 | 1944 豫湘桂 | 最深的低谷 |
| 12 | 1945.8 胜利 | 还乡，尾章按全书 flag 分化结局 |

每章附「史实注」（本章的真实底本与出处线索）——知乎人群最认的形态，也是"用 AI 重访一段历史"的直接证词。

敏感与分寸：南京、饥荒等章以亲历者视角的克制侧写为准则，不消费苦难、不渲染暴行场面；涉真实人物一律侧写不出场。

## 工程与分期

首发只做《山河十四年》一个故事（2026-07-02 用户裁定）；框架的多故事能力保留在架构里，《漫长告别》等后续故事不在本期范围。

| 阶段 | 内容 | 备注 |
|---|---|---|
| P1 | 框架（schema＋引擎＋UI）＋《山河十四年》完整可玩 | 参赛最小完整品，硬底线 |
| P2 | AI 润色层（扩写＋自由输入） | 按赛期弹性 |

技术栈：Vite + React + TS 纯前端静态站，部署与 365 系列惯例一致。

## 风险与对策

- **文本量是最大风险**（十四年跨度 > 孤岛一年）：每章"底稿→精修"两轮；史实注与调研可部分复用 spy 时期的积累；章数封顶 14。
- 章末汇聚意味着没有真正的路线分岔——用 flag 回响与多结局补偿选择感；若试玩反馈"选择太轻"，再评估在 2～3 个关键章做双支线（结构可控的例外）。
- 两项目参赛规则报名前确认。

## 不做的事

- 无属性、无数值结算、无随机事件池。
- 真实历史人物不出场作可交互 NPC。
- P1 不做知乎 API 接入。
